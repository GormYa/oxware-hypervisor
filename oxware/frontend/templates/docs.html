Sisteme Genel Bakış

OXware'un mimarisi, bileşenleri ve teknoloji yığını hakkında kapsamlı bilgi.

OXware Nedir?

OXware; KVM (Kernel-based Virtual Machine) ve QEMU altyapısı üzerine inşa edilmiş, web tabanlı bir hypervisor yönetim sistemidir. Sunucu üzerinde doğrudan çalışır — herhangi bir dış hypervisor'a bağlanmaz. Sistem kendisi hypervisor'dur.

ℹ️
OXware Type-1 hypervisor mantığıyla çalışır: Linux çekirdeğinin KVM modülü doğrudan donanım üzerinde sanallaştırma sağlar.

OXware Kurulumu

Mevcut bir Ubuntu 22.04 / 24.04 LTS sunucusuna OXware kurulum adımları.

Ön Koşullar

⚠️
BIOS/UEFI üzerinden VT-x (Intel) veya AMD-V (AMD) sanallaştırma aktif olmalıdır. Aksi hâlde VM'ler çalışmaz.
# Sanallaştırma kontrolü
grep -Ec "vmx|svm" /proc/cpuinfo
# 0 çıkarsa BIOS'ta aktif etmeniz gerekir

Otomatik Kurulum (Önerilen)

  1. Ubuntu 22.04 LTS sunucuya SSH ile bağlanın

    Root veya sudo yetkili kullanıcı ile giriş yapın.

  2. OXware dosyalarını sunucuya kopyalayın

    scp -r ./OXware root@<ip>:~/OXware

  3. Kurulum scriptini çalıştırın

    cd ~/OXware
sudo bash install.sh

  4. Tarayıcıdan erişin

    https://<sunucu-ip>:8006

Manuel Kurulum

1. Sistem Güncelleme ve Paketler

sudo apt update && sudo apt upgrade -y
sudo apt install -y qemu-kvm libvirt-daemon-system libvirt-clients \
  bridge-utils virtinst python3 python3-pip python3-venv python3-dev \
  libvirt-dev pkg-config gcc novnc websockify ufw fail2ban openssl curl

2. Kullanıcı Grupları

sudo usermod -aG libvirt $USER
sudo usermod -aG kvm $USER
sudo systemctl enable --now libvirtd

3. Python Ortamı

python3 -m venv /opt/oxware/venv
/opt/oxware/venv/bin/pip install flask flask-socketio flask-jwt-extended \
  flask-cors eventlet libvirt-python psutil cryptography requests

4. Yapılandırma Dosyası

cat > /etc/oxware/oxware.conf << 'EOF'
[server]
host        = 0.0.0.0
port        = 8006
ssl         = true
ssl_cert    = /etc/oxware/ssl/oxware.crt
ssl_key     = /etc/oxware/ssl/oxware.key
secret_key  = $(python3 -c "import secrets; print(secrets.token_hex(32))")
novnc_dir   = /usr/share/novnc

[storage]
data_dir     = /var/lib/oxware
iso_dir      = /var/lib/oxware/isos
disk_dir     = /var/lib/oxware/disks
backup_dir   = /var/lib/oxware/backups
template_dir = /var/lib/oxware/templates

[vnc]
start_port     = 5900
end_port       = 5999
websocket_port = 6080

[libvirt]
uri = qemu:///system

[logging]
log_dir = /var/log/oxware
level   = INFO
EOF

5. SSL Sertifikası

sudo mkdir -p /etc/oxware/ssl
sudo openssl req -x509 -nodes -days 3650 -newkey rsa:2048 \
  -keyout /etc/oxware/ssl/oxware.key \
  -out    /etc/oxware/ssl/oxware.crt \
  -subj   "/CN=$(hostname -I | awk '{print $1}')"

6. systemd Servisi

cat > /etc/systemd/system/oxware.service << 'EOF'
[Unit]
Description=OXware Hypervisor Management Panel
After=network.target libvirtd.service

[Service]
Type=simple
User=root
WorkingDirectory=/root/OXware/oxware/backend
ExecStart=/opt/oxware/venv/bin/python app.py
Restart=on-failure
RestartSec=5
StandardOutput=append:/var/log/oxware/oxware.log
StandardError=append:/var/log/oxware/oxware.log

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now oxware

7. Güvenlik Duvarı

sudo ufw allow 22/tcp
sudo ufw allow 8006/tcp
sudo ufw allow 5900:5999/tcp
sudo ufw allow 6080/tcp
sudo ufw --force enable

İlk Giriş ve Kurulum Sihirbazı

OXware'a ilk kez giriş yaparken izlenecek adımlar.

İlk Erişim

Tarayıcıdan https://<sunucu-ip>:8006 adresine gidin. Self-signed SSL sertifikası nedeniyle uyarı gelecektir:

  • Chrome/Edge: Gelişmiş → Devam et
  • Firefox: Gelişmiş → Riski kabul et ve devam et

Kurulum Sihirbazı

Sistem ilk açılışta /setup sayfasına yönlendirir:

  1. Kullanıcı adı belirleyin

    Harf ve rakamlardan oluşan benzersiz bir kullanıcı adı girin.

  2. Güçlü bir parola girin

    Parola gücü göstergesi yeşile dönünce devam edebilirsiniz. En az 8 karakter önerilir.

  3. Kurulumu Tamamla

    Sistem kimlik bilgilerini AES-256 şifreleyerek /etc/oxware/.auth dosyasına kaydeder ve sizi ana panele yönlendirir.

Parola Sıfırlama

Parolanızı unutursanız SSH ile sunucuya bağlanın:

sudo nano /etc/oxware/.passwd_reset

# Dosya içeriği:
USERNAME=yeni_kullanici
PASSWORD=YeniParola123!
sudo systemctl restart oxware
# Servis başlarken dosyayı okur, uygular ve otomatik olarak siler

JWT Token

Başarılı girişte JWT token localStorage'a kaydedilir. Token 8 saat geçerlidir. Süre dolunca otomatik olarak çıkış yapılır.

Snapshot Sistemi

VM anlık görüntüleri alma, geri yükleme ve yönetme.

Snapshot Nedir?

Snapshot, bir VM'in belirli bir andaki tam durumunu (disk + bellek) kaydeder. İstediğiniz zaman o ana geri dönebilirsiniz.

⚠️
Snapshot geri yüklemesi VM'i o anki durumdan geri döndürür. Geri yükleme sonrası mevcut değişiklikler kaybolur.

Snapshot Alma

  1. Snapshot sayfasına gidin

    Sol menüden Snapshot Yönetimi'ni seçin.

  2. VM seçin

    Üstteki açılır listeden snapshot almak istediğiniz VM'i seçin.

  3. 📷 Snapshot Al

    Butona tıklayın, ad ve açıklama girin, onaylayın.

Geri Yükleme

Snapshot listesindeki ↩ Geri Yükle butonuna tıklayın. Onay kutusunu geçtikten sonra VM otomatik olarak o snapshot anına döner.

libvirt Snapshot Komutu

# Snapshot al
virsh snapshot-create-as <vm-adi> <snap-adi> --description "Açıklama"

# Listele
virsh snapshot-list <vm-adi>

# Geri yükle
virsh snapshot-revert <vm-adi> <snap-adi>

# Sil
virsh snapshot-delete <vm-adi> <snap-adi>

IP Havuzu Yönetimi

VM'lere otomatik IP ataması için IP havuzu tanımlama ve yönetme.

IP Havuzu Nedir?

IP havuzu, VM'lere otomatik olarak atanacak IP adresleri aralığını tanımlar. Yeni bir VM oluşturulduğunda veya otomatik provisioning yapıldığında havuzdan bir sonraki boş IP atanır.

Havuz Oluşturma

POST /api/ippool
{
  "name":     "main-pool",
  "network":  "192.168.100.0/24",
  "gateway":  "192.168.100.1",
  "start_ip": "192.168.100.10",
  "end_ip":   "192.168.100.100"
}

IP Tahsis Mantığı

  1. Önce OXware IP havuzu sorgulanır
  2. Bulunamazsa libvirt DHCP lease tablosuna bakılır
  3. O da yoksa ARP tablosu taranır
  4. Hiçbiri bulunamazsa IP alanı boş kalır

Veri Dosyası

IP havuzu bilgileri /var/lib/oxware/ip_pool.json dosyasında saklanır.

{
  "pools": {
    "main-pool": {
      "network":  "192.168.100.0/24",
      "gateway":  "192.168.100.1",
      "start_ip": "192.168.100.10",
      "end_ip":   "192.168.100.100",
      "allocations": {
        "192.168.100.10": { "vm_id": "uuid...", "vm_name": "web-01" }
      }

  }

VM Konsol (noVNC)

Tarayıcı üzerinden VM ekranına doğrudan erişim.

noVNC Nasıl Çalışır?

Her VM, QEMU tarafından bir VNC sunucusu üzerinden erişilebilir hale getirilir. websockify bu VNC portunu WebSocket protokolüne dönüştürür. noVNC bu WebSocket bağlantısını tarayıcıda görselleştirir.

VM (QEMU) → VNC :590x → websockify :6080 → WebSocket → noVNC (Tarayıcı)

Port Ataması

VM'ler 5900–5999 arasında VNC portlarına atanır. İlk VM 5900, ikincisi 5901 vb.

Güvenlik

⚠️
VNC portları (5900–5999) doğrudan internete açık olmamalıdır. Tüm VNC erişimi noVNC üzerinden JWT doğrulamalı WebSocket ile yapılmalıdır.

AI Ajan Sistemi

OpenRouter, Claude, OpenAI veya Ollama ile akıllı sistem izleme ve anomali tespiti.

AI Ajan Nedir?

Her AI ajan, arka planda belirli aralıklarla sistem metriklerini toplayarak bir dil modeline gönderir. Model yanıtını analiz eder, anormal durumlar varsa bildirim gönderir ve sonuçları loglar.

  -subj   "/CN=$(hostname -I | awk '{print $1}')"
sudo systemctl restart oxware

OXware, kurulumla birlikte iki CLI aracı içerir: ox ve oxupdate.

ox komutu, OXware servisini terminal üzerinden yönetmek için kullanılan ana CLI aracıdır.

ox --help          # Tüm komutları listele

ox --url           # Web arayüz URL'sini göster
ox --config        # Yapılandırma dosyası yolunu göster
ox --update        # oxupdate çalıştır
ox --version       # Sürüm bilgisi

oxupdate — Güncelleme Aracı

oxupdate, OXware'i en son sürüme güncellemek için kullanılır. Root yetkisi gerektirir.

sudo oxupdate     # OXware'i en son sürüme güncelle

oxupdate şu adımları otomatik olarak gerçekleştirir:

  1. OXware servisini durdurur
  2. GitHub'dan (origin/main) en son versiyonu çeker
  3. Python bağımlılıklarını günceller (pip install -r requirements.txt)
  4. Servisi yeniden başlatır ve durumu doğrular
ℹ️
ox --update komutu da oxupdate'i çağırır. İkisi birbirinin eşdeğeridir.
⚠️
Güncelleme için git paketi kurulu olmalıdır. Kurulu değilse: apt-get install git

Çözülen Zaafiyetler

OXware sisteminde tespit edilip kapatılan güvenlik açıklarının kaydı.

🛡️ Güvenlik Yaması Politikası

OXware; CVE Tracker modülü ile stack'ini sürekli izler. Tespit edilen zaafiyetler requirements.txt minimum sürüm güncellemesi + servis başlatma sırasında otomatik pip upgrade ile kapatılır.

# /var/log/oxware/pip-upgrade.log — her servis restartında çalışır
/opt/oxware/venv/bin/pip install --quiet --upgrade urllib3 jinja2 cryptography requests

📋 Kapatılan CVE'ler

ÇÖZÜLDÜ CVE-2026-44431 🟠 HIGH · urllib3

Sorun: urllib3 cross-origin redirect sırasında Authorization, Cookie ve Proxy-Authorization headerlarını yeni hedefe iletiyordu. Credentials sızması riski.

Çözüm: urllib3>=2.3.0 requirements.txt'e eklendi. Servis başlatmada otomatik upgrade.

ÇÖZÜLDÜ CVE-2026-44432 🟠 HIGH · urllib3

Sorun: urllib3 streaming API'de bellek sorununa yol açan hata. Büyük yanıtlarda hafıza sızıntısı riski.

Çözüm: urllib3>=2.3.0 — aynı güncelleme ile kapatıldı.

ÇÖZÜLDÜ CVE-2026-45017 🟠 HIGH · Jinja2

Sorun: Jinja2 FileSystemLoader / CachingFileSystemLoader mutlak path verildiğinde arama dizini dışındaki dosyaları okuyabiliyordu. {% raw %}{% include %}{% endraw %} / {% raw %}{% render %}{% endraw %} üzerinden path traversal.

Çözüm: Jinja2>=3.1.6 requirements.txt'e eklendi. Servis başlatmada otomatik upgrade.

ÇÖZÜLDÜ cryptography paketi Önleyici güncelleme

Sorun: Eski cryptography==42.0.8 pinlenmiş sürümde bilinen çeşitli zayıflıklar.

Çözüm: cryptography>=43.0.0 minimum versiyona yükseltildi.

ÇÖZÜLDÜ ISO Installer — Shell Injection 🔴 CRITICAL · build/installer/install.py

Sorun: echo 'user:pass' | chpasswd komutu şifrede tek tırnak (') karakteri olduğunda shell injection'a açıktı. Özel karakterli şifreler kurulumu bozuyor ya da komut enjeksiyonuna yol açabiliyordu.

Çözüm: subprocess.run(["chroot", ..., "chpasswd"], input="user:pass\n", text=True) — shell üzerinden geçmeden doğrudan stdin ile besleme.

ÇÖZÜLDÜ ISO Installer — Yanlış GRUB Paketi 🟠 HIGH · build/installer/install.py

Sorun: Ubuntu 22.04 kurulumunda grub2-common paketi aranıyordu; bu paket Debian'a özgüdür, Ubuntu'da yoktur. Kurulum boot yükleyicisi olmadan tamamlanıyordu.

Çözüm: grub-common (Ubuntu) olarak düzeltildi. grub-pc, grub-efi-amd64, shim-signed de eklendi.
